# 需求文件 — LINE Notion 生活管理 Bot

> 最後更新：2026-05-22　版本：1.3.0

---

## 1. 專案目標

讓使用者和夥伴在 LINE 上，用自然語言隨時記錄採購需求與代辦事項，Bot 自動即時將資訊同步到 Notion，形成可共同瀏覽、追蹤狀態的清單。

---

## 2. 使用者故事

### 採購清單

| # | 需求 | 驗收標準 |
|---|------|---------|
| U1 | 在 LINE 輸入「沙發 預算20000」，Bot 自動記錄到採購清單 | Notion 新增一列，品項=沙發，條件=預算20000 |
| U2 | 同一品項再次輸入新條件，條件自動累加不建新列 | 同列條件欄更新為「舊條件；新條件」 |
| U3 | 查看採購清單 | 輸入「清單」，Bot 回覆最近 10 筆記錄含狀態 |

| U5 | 標記某品項放棄購買 | 輸入「已放棄 沙發」，Notion 狀態改為已放棄 |
| U6 | 刪除採購記錄 | 輸入「刪除 沙發」，Notion 對應列封存移除 |

## 3. 功能規格

### 3.1 訊息解析

Bot 接收 LINE 文字訊息後依下列優先順序處理：

1. **指令比對**（高優先）：符合指令格式者執行對應操作，不觸發記錄
2. **代辦指令**：訊息以 `待辦`、`待辦：`、`todo`、`todo:` 開頭且後接內容 → 記錄到代辦資料庫
3. **採購解析**（fallback）：其餘訊息解析為採購品項
   - 若包含已知家電／家具關鍵字 → 以關鍵字為品項，其餘為條件
   - 否則 → 第一個空白前的字詞為品項，其餘為條件

### 3.2 採購指令

| 指令格式 | 說明 |
|---------|------|
| `{品項} {條件}` | 記錄採購品項（條件可省略） |
| `清單` / `列表` / `list` | 顯示最近 10 筆採購記錄 |
| `已買 {品項}` | 最新符合品項標記為已購買 |
| `已放棄 {品項}` | 最新符合品項標記為已放棄 |
| `刪除 {品項}` | 封存採購或代辦記錄（優先採購，次查代辦） |

### 3.3 代辦指令

| 指令格式 | 說明 |
|---------|------|
| `待辦 {任務}` 或 `待辦：{任務}` | 記錄代辦，第一個空白後的第一個詞為任務名稱 |
| `待辦 {任務} {備註}` | 記錄代辦並附帶備註 |
| `todo {任務} {備註}` | 同上（英文格式） |
| `待辦清單` / `todo清單` / `todolist` | 顯示最近 10 筆代辦記錄 |
| `完成 {任務}` | 最新符合任務標記為已完成 |
| `取消 {任務}` | 最新符合任務標記為已取消 |

### 3.4 Notion 採購清單資料庫欄位

| 欄位名稱 | 類型 | 說明 |
|---------|------|------|
| 品項 | Title | 家電/家具名稱（自動解析） |
| 條件說明 | Rich Text | 累加的預算、尺寸、顏色等條件（以「；」分隔） |
| 原始訊息 | Rich Text | 使用者輸入的完整原文 |
| 加入者 | Rich Text | LINE 顯示名稱 |
| 狀態 | Select | 待購買 / 已購買 / 已放棄 |
| 新增日期 | Date | 記錄建立時間（UTC+8） |

### 3.5 Notion 代辦事項資料庫欄位

| 欄位名稱 | 類型 | 說明 |
|---------|------|------|
| 任務 | Title | 代辦任務名稱 |
| 備註 | Rich Text | 補充說明（可省略） |
| 原始訊息 | Rich Text | 使用者輸入的完整原文 |
| 加入者 | Rich Text | LINE 顯示名稱 |
| 狀態 | Select | 待辦 / 已完成 / 已取消 |
| 新增日期 | Date | 記錄建立時間（UTC+8） |

## 4. 非功能需求

| 項目 | 規格 |
|------|------|
| 回應時間 | Bot 應在 3 秒內回覆（LINE Webhook timeout 限制） |
| 可用性 | Render Free 方案接受閒置 15 分鐘後冷啟動延遲（約 30~60 秒） |
| 安全性 | 使用 LINE Middleware 驗證 Webhook 簽章；金鑰存放於環境變數，不進版控 |
| 去重 | 以 LINE message ID 做記憶體去重，防止 Webhook retry 重複寫入 |
| 費用 | 全程使用免費方案（LINE Free / Notion Free / Render Free） |

---

## 5. 已知限制

- 採購解析依賴關鍵字清單，品項不在清單中則以第一個字詞為品項名稱，可能不準確
- `已買` / `已放棄` / `完成` / `取消` 指令只更新最新一筆符合記錄
- `刪除` 指令優先搜尋採購清單，再搜尋代辦，同名時只刪除最新一筆
- Render Free 方案閒置後冷啟動約 30~60 秒，期間 LINE 驗證可能失敗（重新 Verify 即可）
- 去重 Set 存於記憶體，重啟後清空（正常情況下 Webhook retry 在重啟前就會發生）
