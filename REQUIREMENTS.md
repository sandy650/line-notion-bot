# 需求文件 — LINE Notion 採購清單 Bot

> 最後更新：2026-05-16

---

## 1. 專案目標

讓使用者和夥伴在 LINE 上，隨時輸入任何家電、家具及對應條件說明，Bot 自動即時將資訊記錄到 Notion 資料庫，形成一份可共同瀏覽、追蹤狀態的採購清單。

---

## 2. 使用者故事

| # | 身份 | 需求 | 驗收標準 |
|---|------|------|---------|
| U1 | 使用者 | 在 LINE 輸入「冰箱 預算15000內」，Bot 自動記錄到 Notion | Notion 新增一列，品項=冰箱，條件=預算15000內 |
| U2 | 使用者 | 輸入自由格式文字，Bot 嘗試解析品項與條件 | Bot 回覆確認訊息，顯示解析結果 |
| U3 | 使用者 | 查看目前清單 | 輸入「清單」，Bot 回覆最近10筆記錄含狀態 |
| U4 | 使用者 | 標記某品項已購買 | 輸入「已買 冰箱」，Notion 對應列狀態改為已購買 |
| U5 | 使用者 | 標記某品項放棄購買 | 輸入「已放棄 沙發」，Notion 對應列狀態改為已放棄 |
| U6 | 使用者 | 查看使用說明 | 輸入「說明」，Bot 回覆使用說明 |
| U7 | 使用者 | 在群組或個人對話皆可使用 | Bot 在群組和 1 對 1 對話中行為一致 |

---

## 3. 功能規格

### 3.1 訊息解析（方案 C — 自由輸入）

- Bot 接收 LINE 文字訊息後，**嘗試解析所有訊息**
- 解析優先順序：
  1. 若訊息包含**已知家電/家具關鍵字**，以該關鍵字為品項，其餘為條件
  2. 若無已知關鍵字，以**第一個空白前的字詞**為品項，其餘為條件
- 例外：指令（清單、說明、已買、已放棄）不觸發記錄流程

### 3.2 支援指令

| 指令 | 說明 |
|------|------|
| `清單` / `列表` / `list` | 顯示最近 10 筆記錄 |
| `說明` / `help` / `幫助` | 顯示使用說明 |
| `已買 {品項}` | 將最新符合品項標記為「已購買」 |
| `已放棄 {品項}` | 將最新符合品項標記為「已放棄」 |

### 3.3 Notion 資料庫欄位

| 欄位名稱 | 類型 | 說明 |
|---------|------|------|
| 品項 | Title | 家電/家具名稱（自動解析） |
| 條件說明 | Rich Text | 預算、尺寸、顏色等條件 |
| 原始訊息 | Rich Text | 使用者輸入的完整原文 |
| 加入者 | Rich Text | LINE 顯示名稱 |
| 狀態 | Select | 待購買 / 已購買 / 已放棄 |
| 新增日期 | Date | 記錄建立時間（UTC+8 ISO 格式） |

### 3.4 群組與個人對話

- **個人對話**：所有訊息皆觸發解析與記錄
- **群組對話**：所有訊息皆觸發解析與記錄（Bot 需已加入群組）
- 兩者行為一致，加入者欄位會記錄發訊息者的 LINE 顯示名稱

## 4. 系統架構

```
使用者 LINE App
      |
      | (Webhook HTTP POST)
      v
Express.js 伺服器（Node.js）
      |
      |-- lineHandler.js   處理 LINE 事件、取得發訊者名稱
      |-- messageParser.js 解析品項與條件
      |-- notionService.js 操作 Notion API
      |
      v
Notion 資料庫
```

### 技術選型

| 項目 | 選擇 | 理由 |
|------|------|------|
| 執行環境 | Node.js | LINE SDK 官方支援 |
| Web 框架 | Express.js | 輕量，適合 Webhook |
| LINE SDK | @line/bot-sdk v9 | 官方 SDK |
| Notion SDK | @notionhq/client v2 | 官方 SDK |
| 部署平台 | Railway（免費方案） | 穩定、免費、支援 HTTPS |

---

## 5. 非功能需求

| 項目 | 規格 |
|------|------|
| 回應時間 | Bot 應在 3 秒內回覆（LINE timeout 限制） |
| 可用性 | 免費方案接受偶發性冷啟動延遲 |
| 安全性 | 使用 LINE Middleware 驗證 Webhook 簽章，金鑰存放於環境變數 |
| 費用 | 全程使用免費方案（LINE / Notion / Railway） |

---

## 6. 已知限制

- 自由輸入解析依賴關鍵字清單，若品項不在清單中則以第一個字詞為品項名稱（可能不準確）
- 「已買」/「已放棄」指令更新最新一筆符合記錄，同名品項多筆時只更新最新一筆
- Railway 免費方案可能有每月使用時數限制，需注意
